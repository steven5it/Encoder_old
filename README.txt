Project 3 Encoder

- Huffman algorithm was taken from http://rosettacode.org/wiki/Huffman_coding#Java and modified to fit the project.
- build a encoding using the Huffman Tree based on the frequencies of characters input file given
- generate a random text file (of x input characters) based on the probabilities of characters
- encode the text with the Huffman coding devised previously (be able to decode as well to get original text file)
- repeat procedure except this time, use encoding based on two consecutive characters
- compute the entropy of each encoding and then compare results


Analysis: in most cases, the encoding using 2 symbols performs better (fewer bits/symbol) than using 1 symbol frequencies.
